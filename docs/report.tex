\section{Introducción}
Esta práctica desarrolla una actividad con JADE centrada en un caso de uso sencillo pero completo: un agente con interfaz Swing que dialoga con otro agente para demostrar envío y recepción de mensajes dentro de una plataforma multiagente.

La memoria presenta primero la instalación del entorno de ejecución y, a continuación, el diseño del ejercicio, la arquitectura de los agentes, el funcionamiento general y las pruebas realizadas. El objetivo es dejar un resultado claro, reproducible y fácil de revisar.

\section{Instalación de JADE}
\subsection{Opción recomendada hoy}
La documentación oficial actual de JADE indica que la bifurcación \emph{Jade 4.5.3} puede integrarse directamente mediante Maven usando JitPack \cite{jade-download}. Esta opción es la que se ha documentado en el proyecto porque encaja mejor con una entrega reproducible en un repositorio moderno.

El \texttt{pom.xml} del ejercicio añade la dependencia y el repositorio necesarios. La configuración empleada fuerza Java 8 como nivel de compilación para mantener compatibilidad con la versión pedida en la práctica.

\begin{lstlisting}[style=javaStyle, caption={Fragmento esencial del `pom.xml`.}]
<repositories>
  <repository>
    <id>jitpack.io</id>
    <url>https://jitpack.io</url>
  </repository>
</repositories>

<dependency>
  <groupId>com.gitlab.jade-project</groupId>
  <artifactId>jade</artifactId>
  <version>master-SNAPSHOT</version>
</dependency>
\end{lstlisting}

\subsection{Nota sobre el procedimiento clásico}
El enunciado original de la actividad hace referencia al despliegue clásico de JADE mediante ficheros descargados. Esa vía sigue apareciendo en la documentación de arranque del runtime \cite{jade-start}, pero para esta entrega se ha preferido la integración por Maven porque reduce pasos manuales y deja una base más limpia para el proyecto Java.

\subsection{Verificación de requisitos}
La página oficial del proyecto indica que JADE está implementado en Java y requiere una versión de JRE superior a 1.7 \cite{jade-about}. Por eso la solución puede documentarse y compilarse sin problema sobre Java 8, que es el nivel elegido para la actividad.

\section{Desarrollo del ejercicio}
\subsection{Estructura del proyecto}
El repositorio se organiza en dos bloques:

\begin{itemize}[leftmargin=*]
  \item \texttt{java/}, con el proyecto Maven y los agentes JADE;
  \item \texttt{docs/}, con la memoria, la bibliografía y el \texttt{Makefile}.
\end{itemize}

La decisión de separar código y documentación facilita la evaluación y evita mezclar los ficheros fuente con los artefactos de compilación.

\subsection{Arquitectura de agentes}
El ejercicio adicional se ha reducido a una interacción clara entre dos agentes:

\begin{itemize}[leftmargin=*]
  \item \texttt{ChatClientAgent}, que abre una interfaz Swing para escribir mensajes;
  \item \texttt{ChatResponderAgent}, que recibe mensajes, aplica una respuesta sencilla y la devuelve al cliente.
\end{itemize}

El contenedor principal arranca ambos agentes y activa la consola administrativa de JADE para poder supervisar el estado del sistema desde el inicio.

\subsection{Cliente Swing}
La clase \texttt{ChatWindow} encapsula la interfaz gráfica. Incluye:

\begin{itemize}[leftmargin=*]
  \item área de transcripción para mensajes enviados y recibidos;
  \item campo de texto de entrada;
  \item botón de envío;
  \item ayuda breve con los comandos soportados.
\end{itemize}

El cliente no envía mensajes directamente desde la interfaz gráfica. En su lugar, los encola y deja que el agente los procese en su propio ciclo de comportamiento, lo que evita mezclar la lógica de UI con la lógica de mensajería.

\subsection{Agente respondedor}
El agente respondedor escucha mensajes ACL y genera respuestas simples. Si recibe el texto \texttt{hora} o \texttt{/hora}, devuelve la hora actual. Si recibe \texttt{ayuda} o \texttt{/ayuda}, muestra los comandos disponibles. En cualquier otro caso, contesta con un eco del mensaje.

\section{Funcionamiento}
La ejecución se realiza con Maven:

\begin{lstlisting}[style=javaStyle, caption={Comando de ejecución del ejercicio.}]
cd java
mvn exec:java -Dexec.mainClass=com.unir.jade.chat.Main
\end{lstlisting}

Al arrancar, se observa la consola administrativa de JADE y la ventana del cliente. La interacción básica prevista es:

\begin{enumerate}[leftmargin=*]
  \item escribir un mensaje en la ventana Swing;
  \item pulsar \emph{Enviar};
  \item comprobar que el agente respondedor procesa el texto;
  \item verificar la respuesta en la transcripción del cliente.
\end{enumerate}
